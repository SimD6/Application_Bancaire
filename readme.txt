Ce projet a pour but de présenter une application web créée par Simon Goudeau. 

Fonctionnalité de l'application : 
- Récupérer les informations des transactions des différents comptes bancaires de l'utilisateur
- Catégoriser les transactions pour budgetiser les dépenses du mois
- Provisionner certaines dépenses pour mieux les anticiper

Pourquoi cette application ? 
1. Aucune application sur le marché propose de provisionner ses dépenses annuelles (les impôts, l'achat d'une maison, etc)
2. Aucune application sur le marché propose une interface adaptée au suivi quotidien de ses dépenses.
3. Aucune application sur le marché permet un grand niveau de personnalisation (nombre de catégories, logo des catégories, couleurs, etc.)
4. Aussi et surtout, parce que je sortais de formation et j'avais très envi de mettre en pratique tout ce que je venais d'apprendre sur la réalisation d'application cloud.

Technologie utilisée : 
- Firebase (service d'authentification, base de donnée en temps réelle) -> permet notamment d'avoir une base sécurisée
- Nordigen (ou Open Bankin) pour la récupération des informations sur les transactions 
- Fast Api pour le backend
- React pour le frontend
